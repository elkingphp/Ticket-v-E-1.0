Technical Documentation / Database Schema

Database Schema

Table: tickets.tickets

Column Type Constraints Description
id BIGINT PK, Serial Internal relational ID
uuid UUID Unique, Indexed Publicly exposed identifier
ticket_number VARCHAR Unique Human-readable label
user_id BIGINT FK (users.id) Ticket owner
metadata JSONB - Dynamic attributes storage
status_id BIGINT FK Current state reference

Design Rationale

  • PostgreSQL Schema Separation: Isolates ticket data into a dedicated namespace to avoid collision and improve backup strategies.
  • JSONB Storage: Used for extensible metadata, allowing different ticket categories to have specific data fields without schema migrations.
  • Foreign Key Integrity: Strict constraints on user_id and status_id to prevent orphaned records.
