Technical Documentation / Domain Design

Domain Design

Core Entities

  • Ticket: The Root Aggregate. Controls state and holds global metadata.
  • Thread: Individual communication points (User Reply, Agent Comment, Internal Note).
  • Categories/Stages: Classification entities providing structural semantics.

Business Rules

Design follows the Rule-Driven State Transition pattern:

  • A ticket cannot be closed without at least one solution thread.
  • Only users with agent.tickets role can claim an unassigned ticket.
  • Transitioning to "Resolved" triggers an automated notification to the owner.

ER Explanation

The system utilizes a One-to-Many relationship between Ticket and Thread. The Ticket entity acts as a gateway for all associated activity, ensuring consistency across the entire communication lifecycle.
