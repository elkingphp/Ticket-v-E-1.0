Technical Documentation / Application Layer

Application Layer

The Application Layer acts as the orchestrator for the Tickets Module. It defines the use cases of the module, receives input from the interfaces (Controllers/APIs), validates the input, and delegates the domain logic to the underlying Domain Layer services.

Core Services

The primary business workflows are exposed through dedicated service classes to ensure a clear separation of concerns.

TicketService

This service serves as the core entry point for managing ticket lifecycles. It orchestrates the creation, updating, assignment, and resolution of tickets. It interacts heavily with repositories to persist domain state changes.

TicketActivityService

Used specifically for maintaining the audit trail and timeline of a ticket. It automatically captures any modification on a ticket and logs the actor, the action, and the related changes.

Controllers

Communication with the front-end features (like the Velzon UI) or external API clients is handled by standard Laravel controllers. These controllers mostly resolve the TicketService and map HTTP requests properties to Service Data Transfer Objects (DTOs) or arguments.

  • TicketController: Handles standard web requests for the ticket management UI.
  • TicketApiController: Manages stateless JSON-based payload exchanges for remote access or decoupled front-ends.

Form Requests (Validation)

Laravel Form Requests are utilized strictly for validating inputs before they reach the controller logic. They define validation rules and authorize the incoming request.

public function rules() {
    return [
        'subject' => 'required|string|max:255',
        'category_id' => 'required|exists:ticket_categories,id',
        'priority' => 'required|in:low,medium,high,critical',
        'content' => 'required|string'
    ];
}

API Resources

When returning data to API clients, Laravel API Resources (e.g., TicketResource) are used to uniformly format and encapsulate domain models into standardized JSON structures, ensuring breaking changes to the database layer don't propagate to API consumers.
