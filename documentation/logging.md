# نظام الأرشفة والتسجيل (Audit Logs)

يعد نظام الأرشفة أحد الركائز الأمنية الأساسية في هذا الإطار البرمجي.

## آلية العمل
1. **الالتقاط التلقائي**: عبر سمة (Trait) `LogsActivity` المضافة للموديلات، يتم رصد أي تغيير في البيانات.
2. **المعالجة الخلفية**: يتم إرسال بيانات الحدث إلى طابور Redis (`ProcessAuditLog` job).
3. **التخزين**: يتم حفظ السجل في قاعدة بيانات PostgreSQL مع المعلومات التالية:
   * من قام بالعمل (User ID).
   * نوع العملية (Event).
   * البيانات التي تغيرت (Request Data).
   * عنوان الـ IP والـ Browser Agent.
   * الموديول الذي حدث فيه التغيير.

## الأمان والخصوصية
* **تغطية البيانات (Masking)**: يتم البحث آلياً عن أي حقول حساسة (مثل password, token) واستبدالها بنص ثابت قبل الحفظ.
* **التشفير**: البيانات المخزنة كـ JSONB تكون محمية ومفهرسة لسرعة البحث.

## تنظيف السجلات
هناك أمر Artisan مجدول يعمل يومياً لتنظيف السجلات القديمة بناءً على الإعدادات:
`php artisan audit:cleanup`
*يتم تحديد عدد أيام الاحتفاظ بالسجلات من إعدادات النظام.*
