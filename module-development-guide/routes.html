إدارة المسارات (Routes)

ربط العناوين بالمتحكمات داخل الموديول

الرئيسية إنشاء موديول الهيكل البرمجي المسارات (Routes) المتحكمات (Controllers) الصلاحيات الإعدادات مثال CRUD واجهة Velzon English

أين تجد ملفات المسارات؟

توجد المسارات في مجلد Routes داخل كل موديول، مقسمة إلى:

  • web.php: للواجهات الرسومية (تتضمن الجلسات و CSRF).
  • api.php: للواجهات البرمجية (تتضمن الـ JSON والـ Stateless auth).

مثال لإضافة مسار ويب:

use Illuminate\Support\Facades\Route;
use Modules\Inventory\Http\Controllers\ItemController;

Route::prefix('inventory')->group(function() {
    Route::get('/', [ItemController::class, 'index'])->name('inventory.index');
    Route::post('/store', [ItemController::class, 'store'])->name('inventory.store');
});

تطبيق Middleware الصلاحيات:

لحماية المسار وتفعيله فقط لمن لديهم صلاحية معينة:

Route::middleware(['auth', 'can:inventory.view'])->group(function() {
    Route::get('/inventory', [ItemController::class, 'index']);
});

حماية الموديول كاملاً

نظامنا يوفر middleware خاص للتحقق من أن الموديول نشط (Enabled) قبل الدخول:

Route::middleware(['module_status:Inventory'])->group(function() {
    // كل المسارات هنا ستتوقف إذا تم تعطيل موديول المخزون
});
تنبيه: تأكد دائماً من استخدام الأسماء (Named Routes) لتسهيل الربط في الواجهات، مثل inventory.list.
