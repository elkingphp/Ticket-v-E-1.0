Why use layers?

The goal is to separate "what the code does" from "how it reaches data" and "how it responds to the browser."

1. Domain Layer (The Heart)

Contains core business logic and rules that do not change:

  • Models: Data definition and relationships.
  • Interfaces: Contracts that Repositories adhere to.
  • Events: Events fired by the module.

2. Application Layer (The Engine)

Controls workflow operations:

  • Services: Aggregating complex operations in one place.
  • Policies: Determining who can do what.
  • Jobs: Background tasks.

3. Infrastructure Layer (The Store)

Deals with external technology (Database, Files):

  • Repositories: Implementing SQL queries via interfaces.
  • Database: Migrations and seeders.

4. Http Layer (The Interface)

Connection point with the user:

  • Controllers: Receive requests and direct them to services.
  • Requests: Validation of input data.
Golden Rule: The Controller should be "Thin," and the Service is the one doing the heavy lifting.
