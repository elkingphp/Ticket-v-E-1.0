How to Define New Permissions

Our system adds a module column to the permissions table for easy separation and source tracking.

1. In Migration File

It is recommended to add the basic permissions for the module upon installation:

use Modules\Core\Domain\Models\Permission;

public function up()
{
    Permission::findOrCreate('inventory.view', 'web', 'Inventory');
    Permission::findOrCreate('inventory.create', 'web', 'Inventory');
}

2. Create Policy

Create a Policy file in Application/Policies:

class InventoryPolicy
{
    public function view(User $user) {
        return $user->hasPermissionTo('inventory.view');
    }
}

3. Register Policy

In Providers/InventoryServiceProvider.php:

public function boot()
{
    Gate::policy(InventoryItem::class, InventoryPolicy::class);
}

4. Usage in View

@can('inventory.create')
    <button>Add New Item</button>
@endcan
Feature: When disabling or deleting a module, the system automatically cleans up associated permissions.
