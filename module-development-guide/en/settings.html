Using SettingsService

The Core module provides a central service for managing settings stored in the database and cached in Redis.

1. Saving a New Setting

You can do this in a Seeder or Controller:

use Modules\Core\Application\Services\SettingsService;

public function __construct(SettingsService $settings) {
    $this->settings = $settings;
}

// Save encrypted setting
$this->settings->set('api_key', '12345', 'Inventory', 'string', true);

2. Fetching a Setting

$val = $this->settings->get('api_key', 'default_value');

Technical Features:

  • Singleton: Settings are loaded only once per Request.
  • Flash Cache: Rapid memory (Redis) is updated immediately upon any change.
  • Auto-Encryption: Sensitive values are automatically encrypted with AES-256 if is_encrypted = true.

Supported Data Types:

string, integer, boolean, array, json

Tip: Use settings for values changed by the admin (like enabling a feature or linking an external API), not for storing constantly changing data.
