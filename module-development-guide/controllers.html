المتحكمات (Controllers)

نقطة الانطلاق لمعالجة الطلبات

الرئيسية إنشاء موديول الهيكل البرمجي المسارات (Routes) المتحكمات (Controllers) الصلاحيات الإعدادات مثال CRUD واجهة Velzon English

أين توضع المتحكمات؟

توضع المتحكمات في مجلد Http/Controllers داخل الموديول.

القاعدة الذهبية

المتحكم لا يجب أن يحتوي على منطق معقد أو استعلامات DB مباشرة. يجب أن يستدعي "الخدمة" (Service).

namespace Modules\Inventory\Http\Controllers;

use Modules\Core\Http\Controllers\BaseController;
use Modules\Inventory\Application\Services\ItemService;
use Modules\Inventory\Http\Requests\SaveItemRequest;

class ItemController extends BaseController
{
    protected $itemService;

    public function __construct(ItemService $service)
    {
        $this->itemService = $service;
    }

    public function store(SaveItemRequest $request) 
    {
        // 1. التحقق من الصلاحيات (اختياري هنا، يفضل في الـ Policy)
        $this->authorize('create', Item::class);
        
        // 2. معالجة البيانات عبر الخدمة
        $item = $this->itemService->createItem($request->validated());

        // 3. العودة بالرد
        return redirect()->back()->with('success', 'تمت الإضافة');
    }
}

استخدام Form Requests

دائماً استقبل البيانات عبر FormRequest خاص موجود في Http/Requests لضمان أمان المدخلات قبل وصولها للمتحكم.

ملاحظة: ورث دائماً من BaseController الموجود في موديول Core للحصول على ميزات توحيد الاستجابة.
