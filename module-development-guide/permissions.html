نظام الصلاحيات الموديولي

ربط الوظائف بأدوار المستخدمين

الرئيسية إنشاء موديول الهيكل البرمجي المسارات (Routes) المتحكمات (Controllers) الصلاحيات الإعدادات مثال CRUD واجهة Velzon English

كيفية تعريف صلاحيات جديدة

نظامنا يضيف عمود module لجدول الصلاحيات لسهولة التقسيم وتتبع المصدر.

1. في ملف الهجرة (Migration)

ينصح بإضافة الصلاحيات الأساسية للموديول عند تثبيته:

use Modules\Core\Domain\Models\Permission;

public function up()
{
    Permission::findOrCreate('inventory.view', 'web', 'Inventory');
    Permission::findOrCreate('inventory.create', 'web', 'Inventory');
}

2. إنشاء الـ Policy

أنشئ ملف Policy في Application/Policies:

class InventoryPolicy
{
    public function view(User $user) {
        return $user->hasPermissionTo('inventory.view');
    }
}

3. تسجيل الـ Policy

في Providers/InventoryServiceProvider.php:

public function boot()
{
    Gate::policy(InventoryItem::class, InventoryPolicy::class);
}

4. الاستخدام في الـ View

@can('inventory.create')
    <button>إضافة صنف جديد</button>
@endcan
ميزة: عند تعطيل الموديول أو حذفه، يقوم النظام تلقائياً بتنظيف الصلاحيات المرتبطة به.
