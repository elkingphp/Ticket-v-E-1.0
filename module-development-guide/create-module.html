إنشاء موديول جديد (الخريطة التنفيذية)

دليل تنفيذي خطوة بخطوة لإنشاء موديول متوافق 100% مع البنية التحتية

الرئيسية إنشاء موديول الهيكل البرمجي المسارات (Routes) المتحكمات (Controllers) الصلاحيات الإعدادات مثال CRUD واجهة Velzon

تهدف هذه الوثيقة العملية (Executable Documentation) إلى تقديم خطوات خطية ودقيقة، إذا قام المطور باتباعها سينتج موديول صحيح فنياً ومتصل بنظام Audit Log ونظام الـ DomainGraph تلقائياً.

التحضير والتوليد (Module Initialization) الخطوة 01

الهدف: تجهيز بيئة العمل وإنشاء الهيكل الأساسي للوحدة النمطية باستخدام حزمة nWidart.

الأمر:
php artisan module:make Inventory
✅ التحقق:
تأكد من إنشاء مجلد app/Modules/Inventory وتسجيل الموديول بحالة true في ملف modules_statuses.json.
تهيئة المعمارية (Folder Structure) الخطوة 02

الهدف: فرض حدود الطبقات المؤسسية (Layered Architecture) بإنشاء المجلدات الناقصة لمنع خلط الأكواد.

الأمر:
mkdir -p app/Modules/Inventory/{Models,Services,Repositories,Http/Requests,Policies,Events,Listeners,Contracts,Graph}
تحذير: يُمنع وضع منطق الأعمال (Business Logic) مباشرة داخل مجلد الـ Http، يجب تفريغه داخل الـ Services.
قاعدة البيانات (Database Migration) الخطوة 03

الهدف: تعريف جداول البيانات مع الإلزام بإضافة أعمدة التدقيق (Audit) والحذف المنطقي.

الأمر:
php artisan module:make-migration create_inventory_items_table Inventory
الكود:
Schema::create('inventory_items', function (Blueprint $table) { $table->id(); $table->uuid('uuid')->unique(); // إلزامي للترابط المبهم $table->string('name'); // أعمدة التدقيق الإلزامية $table->foreignId('created_by')->nullable()->constrained('users'); $table->foreignId('updated_by')->nullable()->constrained('users'); $table->timestamps(); $table->softDeletes(); // إلزامي للحفاظ على تاريخ البيانات });
✅ التحقق:
تشغيل أمر php artisan migrate دون وجود أخطاء صياغة SQL.
نموذج البيانات (Eloquent Model) الخطوة 04

الهدف: تكوين الكيان (Model) ليدعم الـ Traits المؤسسية أوتوماتيكياً.

الملف: app/Modules/Inventory/Models/InventoryItem.php
namespace Modules\Inventory\Models; use Illuminate\Database\Eloquent\Model; use Illuminate\Database\Eloquent\SoftDeletes; use App\Traits\HasUuid; use App\Traits\LogsActivity; class InventoryItem extends Model { use HasUuid, SoftDeletes, LogsActivity; protected $guarded = ['id', 'uuid', 'created_by']; protected $casts = [ 'created_at' => 'datetime', ]; }
طبقة الخدمة والاستعلامات (Repository & Service) الخطوة 05

الهدف: عزل طبقة قاعدة البيانات في Repository وتجميع العمليات المركزية داخل الـ Service.

ملف المستودع Repository:
namespace Modules\Inventory\Repositories; use Modules\Inventory\Models\InventoryItem; class InventoryRepository { public function create(array $data): InventoryItem { return InventoryItem::create($data); } }
ملف الخدمة Service:
namespace Modules\Inventory\Services; use Modules\Inventory\Repositories\InventoryRepository; use Illuminate\Support\Facades\DB; class InventoryService { public function __construct(private InventoryRepository $repo) {} public function createItem(array $data) { return DB::transaction(function () use ($data) { return $this->repo->create($data); // Operations happen safely here }); } }
المسارات و المتحكمات (Routes & Controllers) الخطوة 06

الهدف: بناء واجهة الاستقبال HTTP.

ملف المسار api.php:
Route::middleware(['auth:sanctum'])->prefix('api/inventory')->group(function () { Route::post('/', [InventoryController::class, 'store']); });
ملف المتحكم InventoryController.php:
public function store(StoreInventoryRequest $request, InventoryService $service) { // المتحكم يقوم بالتوظيف فقط، لا يحتوي على Business Logic $item = $service->createItem($request->validated()); return response()->json($item, 201); }
التسجيل بالـ DomainGraph (مهم جداً) الخطوة 07

الهدف: تعريف الموديول للنظام بأكمله لربطه بالبيانات المرجعية ولتفعيل البحث العام.

الملف: إنشاء كلاس للترابط InventoryDomainGraphBuilder.php
namespace Modules\Inventory\Graph; use App\Contracts\DomainGraphNode; use App\Services\DomainGraph; class InventoryDomainGraphBuilder implements DomainGraphNode { public function registerNodes(DomainGraph $graph): void { $graph->addNode('inventory', [ 'model' => \Modules\Inventory\Models\InventoryItem::class, 'dependencies' => ['users'] ]); } }
✅ التحقق:
تسجيل هذا الكلاس ضمن مزود الخدمة، واستدعاء الأمر php artisan domain:graph:verify.
مراقبة السلوك (Audit Integration) الخطوة 08

الهدف: تخزين أثر الموديول في نظام التدقيق الموحد.

الكود (يُستخدم داخل الـ Service):
\App\Services\AuditLoggerService::log( action: 'created', model: $item, actor: auth()->user(), context: ['ip' => request()->ip()] );
تمت المهمة (Final Flight Pre-Check) الختام

للتأكد أن الموديول سليم بنسبة 100%، راجع القائمة الآتية قبل دمج الكود:

  • أمر Migration ينفّذ بامتياز بدون انهيار قاعدة البيانات.
  • الكائنات (Models) تحتوي على SoftDeletes.
  • تم تكوين المسارات (Routes) محمية تحت طيات تسجيل الدخول.
  • الملحق البرمجي مسجل بنجاح في مخطط DomainGraph.
  • تم استدعاء خدمة التدقيق لتوثيق التحركات (Audit Logging).
  • تحقيق 80%+ اجتياز في الاختبارات الآلية (Pest/PHPUnit).
